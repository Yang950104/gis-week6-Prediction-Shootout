## 研究問題
在極端強降雨事件下，如何結合即時的空間內插雨量、數值地形模型 (DEM) 以及 Google Earth Engine (GEE) 的遙測透水圖資，建立一套低延遲、高準確度的地下停車場淹水預警系統？

## 資料來源
1. 中央氣象署雨量站觀測資料 — CoLife 歷史資料庫 — JSON / 數萬筆
2. 數值地形模型 (DEM) — 內政部地政司 — Raster (20m 解析度)
3. 多光譜衛星影像 (Sentinel-2/Landsat) — Google Earth Engine (GEE) — Raster (雲端獲取)
4. 市區公/私有地下停車場點位 — 縣市政府資料開放平台 — CSV/GeoJSON

## 分析方法
1. **降雨危險度 (Hazard)**：利用觀測站時雨量進行空間內插，產出連續的降雨強度網格。
2. **地形易淹度 (Vulnerability)**：分析 DEM 高程與坡向，篩選出容易匯聚地表逕流的低窪積水區 (Sinks)。
3. **地表逕流係數 (Exposure)**：利用 GEE 自動計算 NDVI/NDBI 等指標，判釋地表植被覆蓋與不透水層比例。最後將上述三項特徵疊圖 (Spatial Overlay)，計算各停車場的綜合淹水風險值。

## 內插策略
本專案將全面採用 **Ordinary Kriging** 進行空間內插。
原因在於，防汛預警需要高度的可靠性評估。Random Forest 產生的預測邊界過於生硬，且無法評估誤差；Kriging 不僅符合降雨平滑過渡的物理特性，其附帶產生的 Sigma Map (不確定性地圖) 能讓我們標示出「因附近無測站而預警信心不足」的停車場，提供更嚴謹的決策依據。

## Gemini SDK 使用計畫
將 Gemini REST API (gemini-2.5-flash) 整合為「防汛戰術顧問」。當系統偵測到特定地下停車場達到高風險條件（如：降雨網格值大於 50mm + 位於低窪區 + 屬高度不透水層）時，Gemini 將讀取這些空間參數與 Kriging Variance，自動生成兩階段簡訊：
- **信心高時**：直接建議管理員「請立即升起防水閘門，預估地表逕流將大量匯聚」。
- **信心低時**：建議「該區預測誤差較高，請立即派員至車道口目視巡查積水狀況」。

## 風險評估
最可能遇到的技術困難是**多源網格圖資的對位與解析度不匹配**。GEE 輸出的遙測影像、內政部的 DEM 以及我們自行生成的 Kriging 網格，其 CRS (座標系統) 與 Pixel Size 勢必不同，直接疊加會導致空間錯位。
**備案**：開發標準化的 CRS Alignment 與 Resampling 函數，強制將所有 Raster 圖層統一轉換至 EPSG:3826 座標系，並對齊相同的 100m x 100m 網格邊界。
